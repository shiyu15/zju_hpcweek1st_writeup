# Bash Quine
x86 1 核 0.3 GiB 100 分

## 题目描述

在计算机科学中，Quine 是一种非常有趣的自我复制程序，所谓的自我复制，意思是指这个程序不接受任何输入，但是执行时，它能够输出自己的源代码。

为了让同学们了解 Shell 脚本的基本语法，我们将尝试使用 Shell 脚本来编写一个 Quine.

比如下面这个脚本就可以很好地完成任务:
```BASH
cat $0
```
当你执行它的时候，观察它的输出，会发现恰好是自身的内容 cat $0:
```BASH
$ sh quine.sh
cat $0
```
如果你在脚本中输出 $0, 会发现这个变量指代的是这个脚本的文件名，而 cat 会把文件的内容输出出来。所以，cat $0 就借助了 cat 这个工具，把自己的源码输出了出来。

不过，这个脚本并不符合我们对 Quine 的要求，因为我们要求 Quine 不能在运行的时候读取自身的内容。这就需要同学们开动脑筋，也可以参考网络上其他语言的 Quine 实现，来编写一个 Bash 脚本的 Quine。
注: 帮大家试过了，直接提交 cat $0 是过不了此题的 (doge)

## 环境说明
测评环境为一个极简的 Linux 环境，使用的是 BusyBox。 BusyBox 是一个常见的轻量级 Linux 工具集合，它把常用命令（如 cat、awk、sed、echo、tr 等）合并在一个可执行文件中，非常适合嵌入式系统和容器环境。

这意味着你不可以：
使用复杂的 Bash / Zsh / Fish 等解释器的特性, 比如数组和切片，以及它们额外的语法
使用其他编程语言做辅助，如 perl, python 等

相应的，你可以:
使用基础的 POSIX 规范中的 Shell 语法
使用 busybox 自带的常用命令，如 awk, sed, echo 等等
    可以在 busybox 文档查询全部可以使用的命令以及他们的作用

测评程序会比较你的 Bash 脚本的输出和其文件本身的内容是否匹配，文件最末尾的空格会被忽略。
